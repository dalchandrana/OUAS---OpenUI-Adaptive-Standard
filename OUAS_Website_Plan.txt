SECTION 01
Design System
Typography, colour, motion, spacing — the rules everything is built on

Design philosophy
The OUAS website takes direct inspiration from dodopayments.com: near-black backgrounds, sharp editorial typography, generous whitespace, and a developer-first tone that treats visitors as intelligent builders. Every design decision signals: 'this is a serious standard built by people who care about craft.'
Key characteristics borrowed from Dodo Payments and adapted for OUAS: dark background as the default (not an option), code blocks as first-class visual elements, horizontal scrolling logo/partner strips with infinite marquee animation, tabbed feature sections that switch product areas without page reload, and a prominent developer-facing CTA that leads directly to documentation.

Animation system
Four animation categories, each with a precise definition. All animations respect prefers-reduced-motion. No animation exists purely for decoration — every motion communicates a state change or guides attention.
Category
Specification
Entrance animations
Elements fade up 24px on scroll enter. Duration 0.5s, ease-out cubic-bezier(0.16,1,0.3,1). Stagger children by 0.07s. Used on: feature cards, stat numbers, testimonial quotes.
Marquee scroll
Infinite horizontal scroll at 40px/s for logo strips. Two identical rows moving opposite directions (Dodo Payments pattern). CSS animation only, no JS.
Tab transitions
Feature section tab switch: content fades out (0.15s) then new content fades in and slides up 16px (0.3s). No layout shift.
Code highlight
Typing animation on hero code block. Characters appear at 30ms intervals. Cursor blinks at 1Hz. Triggered once on page load.
Glow pulse
Accent elements (CTA button border, active tab) pulse glow opacity 0.15 → 0.35 → 0.15 at 3s cycle. CSS only.
Page transition
Route changes: current page fades out (0.2s), new page fades in from 8px below (0.3s). Framer Motion AnimatePresence.
Number counter
Stats section: numbers count up from 0 when scrolled into view. Duration 1.2s, ease-out. JS IntersectionObserver trigger.
Copy flash
Copy Page CTA: button flashes green for 1.5s after click, text changes to 'Copied!' then resets. CSS transition only.

SECTION 04
Copy Page CTA — Full Specification
The most unique feature of the site — feeds OUAS context to any AI agent

What it is
The Copy Page CTA is a button — appearing on every page — that, when clicked, assembles a complete context document about OUAS and copies it to the user's clipboard. The user can then paste it into any AI agent (Claude, GPT, Cursor, Copilot) and immediately start building with OUAS without the agent needing to search the web or hallucinate the API.
This is directly inspired by the 'llms.txt' movement and context window priming patterns. The difference is that this is interactive — it assembles the right context based on which page the user is currently on.

Context document — what gets copied
The copied text is structured markdown. It varies by page context:
Page
Context assembled
On / (home)
Full OUAS overview: what it is, the 4 packages, withOUAS() usage, Agent API endpoints, key rules (no eval, sealed registry). ~800 tokens.
On /docs/getting-started
Getting started context: install commands, full annotated component example, manifest shape, OUASProvider mounting. ~600 tokens.
On /docs/manifest-spec
Full manifest spec: all fields, all rules, example manifest JSON for MailFlow. ~900 tokens.
On /docs/layout-config-spec
Full layout config spec: all fields, all rules, executive + student + researcher example configs. ~900 tokens.
On /docs/validation
Validator context: all 7 validation steps, all error codes with messages, validation response format. ~500 tokens.
On /docs/agent-api
Agent API context: all 4 endpoints, request/response shapes, auth requirements, rate limits. ~400 tokens.
On /docs/security
Security context: locked components, agent_scope, rate limiting code, ownership enforcement. ~500 tokens.
On /demo
Demo context: MailFlow manifest, 3 example layout configs, agent pipeline explanation. ~700 tokens.

Button placement
Location
Specification
Floating button
Fixed position, bottom-right, z-index 50. On all pages. 48px × 48px square with rounded corners. Icon: clipboard with spark. Tooltip on hover: 'Copy OUAS context for your AI agent'.
Inline version
Full-width banner inside page content. Shows on /docs/getting-started and /demo. Contains: label, description of what gets copied, button, and small note: 'Paste into Claude, GPT-4, or Cursor to start building immediately.'

Compact version at bottom of left sidebar on all /docs/* pages. Small button with label 'Copy page context'.

Implementation
// lib/copy-context.ts
 
const CONTEXT_MAP: Record<string, () => string> = {
  '/': () => buildHomeContext(),
  '/docs/getting-started': () => buildGettingStartedContext(),
  '/docs/manifest-spec': () => buildManifestContext(),
  '/docs/layout-config-spec': () => buildLayoutConfigContext(),
  '/docs/validation': () => buildValidationContext(),
  '/docs/agent-api': () => buildAgentApiContext(),
  '/docs/security': () => buildSecurityContext(),
  '/demo': () => buildDemoContext(),
};
 
export async function copyPageContext(pathname: string): Promise<void> {
  const builder = CONTEXT_MAP[pathname] ?? CONTEXT_MAP['/'];
  const context = builder();
  await navigator.clipboard.writeText(context);
}
 
function buildHomeContext(): string {
  return [
    '# OpenUI Adaptive Standard (OUAS) v1.0',
    '## What it is',
    OUAS_OVERVIEW,
    '## The 4 packages',
    PACKAGES_OVERVIEW,
    '## Quick start',
    QUICKSTART_EXAMPLE,
    '## Key rules',
    KEY_RULES,
    '## Agent API endpoints',
    AGENT_API_SUMMARY,
  ].join('\n\n');
}

SECTION 06
Phased Implementation Plan
10 weeks from blank repo to live public website

Stack decision
Next.js 14+ App Router · TypeScript strict · Tailwind CSS · Framer Motion · MDX · Shiki · Pagefind · Vercel deploy. No exceptions — this stack is optimised for the content mix (marketing + docs + interactive demo).

Phase 1 — Foundation
Weeks 1–2 | Design system + project scaffold

Everything in Phase 1 is invisible to users. No deployed pages yet. This phase produces the design system, the component library, and the project structure that every subsequent phase builds on. Doing this right prevents rewrites.

Week 1 — Project scaffold + design tokens
	•	Initialise Next.js 14 project with App Router, TypeScript strict, Tailwind CSS, ESLint, Prettier
	•	Install Framer Motion, @next/mdx, next-mdx-remote, Shiki, Pagefind
	•	Set up Vercel project, connect GitHub repo, enable preview deployments on every PR
	•	Define all CSS variables (colour palette, spacing scale, typography) in globals.css
	•	Load Syne and JetBrains Mono from Google Fonts via next/font
	•	Configure Tailwind to consume CSS variables (extend theme with var() references)
	•	Set up @/components, @/lib, @/content, @/styles path aliases in tsconfig
