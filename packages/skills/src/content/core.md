# OUAS Core Skills v1.0

## What you are building
You are building an application using the OpenUI Adaptive Standard (OUAS). OUAS is a standard for creating adaptive UIs that strictly separate visual components from layout logic. This allows AI agents to reconfigure layouts dynamically at runtime using JSON Layout Configs.

## Rules — always follow these
RULE-001: Every component must be wrapped with `withOUAS()` before it is OUAS-compatible.
RULE-002: Never hardcode layout in JSX — all layout decisions (positioning, padding, flex direction, margins) belong in a Layout Config.
RULE-003: `withOUAS()` must never modify the component's internal render function.
RULE-004: The manifest is generated by the CLI (`npx ouas generate`) — never write `manifest.ouas.json` by hand.
RULE-005: `OUASProvider` must wrap the entire app at the root level, not per-page.

## Anti-patterns — never do these
- BAD: Putting layout logic inside a React component (e.g., hardcoding `absolute right-4 top-4` classes).
  - GOOD: Using Layout Configs for all positioning.
- BAD: Writing `manifest.ouas.json` by hand.
  - GOOD: Run `npx ouas generate`.
- BAD: Calling the Agent API directly from a component.
  - GOOD: Let the `OUASProvider` manage API communication.
- BAD: Storing Layout Configs in local component state.
- BAD: Using `eval()` anywhere in the renderer.

## Code patterns — copy these exactly

### 1. Annotated withOUAS() component
```tsx
import { withOUAS } from '@ouas/react';

function Button({ variant, children }: { variant: 'primary' | 'secondary', children: React.ReactNode }) {
  // Pure visual component, NO layout classes (no absolute, no padding for layout, no flex parent behavior)
  return (
    <button className={`btn-${variant}`}>
      {children}
    </button>
  );
}

// JSDoc is required for the CLI manifest generator
/**
 * A primary action button. Use this for the main submit action on forms.
 * Supports 'primary' and 'secondary' variants.
 */
export default withOUAS(Button, {
  name: 'Button',
  props: {
    variant: { type: 'string', default: 'primary', options: ['primary', 'secondary'] }
  }
});
```

### 2. OUASProvider mounting
```tsx
import { OUASProvider } from '@ouas/react';
import { initialLayout } from './layout-config.json';

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        <OUASProvider 
          agentEndpoint="/api/agent"
          initialConfig={initialLayout}
        >
          {children}
        </OUASProvider>
      </body>
    </html>
  );
}
```

## Validation checklist — run before you finish
- [ ] Every component shown in the UI has a `withOUAS()` annotation.
- [ ] `npx ouas generate` has been run and `manifest.ouas.json` is current.
- [ ] `OUASProvider` is mounted at app root.
- [ ] No layout decisions exist in component JSX.
